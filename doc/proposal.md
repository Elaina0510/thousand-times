## 一、项目概述

### 1.1 项目目标

构建一套自动化的A股行情分析系统，每日定时从公开数据源抓取股票和行业ETF行情数据，通过技术指标、基本面数据和政策新闻进行综合评分，筛选出**高概率盈利**或**存在特殊风险**的个股和ETF，生成分析报告（含图表），并通过微信推送给用户。

### 1.2 核心原则

- **宁缺毋滥**：只推送综合评分 ≥ 70 分的个股和ETF，无符合条件时推送"今日无推荐"
- **数据可信**：新闻来源需具备可信度，结合历史经验分析政策影响
- **风险提示**：对存在特殊风险的个股和ETF单独标注，帮助用户规避

## 三、功能需求

### 3.1 股票池筛选（过滤规则）

从A股全市场中筛选股票，依次执行以下过滤：

| 序号 | 过滤条件 | 说明 |
|------|----------|------|
| 1 | 剔除市值 < 20亿 | 排除小盘垃圾股 |
| 2 | 剔除名称含"ST" | 排除风险警示股 |
| 3 | 剔除滚动市盈率（PE-TTM）≤ 0 | 排除当前亏损企业 |
| 4 | 剔除上市不满 3 个月 | 排除新股，数据不足 |

过滤完成后，取**总市值排名前1000**的股票进入分析池。

### 3.2 行业ETF分析

同时分析主流行业ETF，ETF与个股采用**相同的筛选与推送逻辑**，独立评分、独立推送。

#### 3.2.1 ETF池

- **大盘宽基ETF**：沪深300ETF、中证500ETF、创业板ETF、科创50ETF等
- **细分行业ETF**：半导体ETF、新能源ETF、医药ETF、消费ETF、军工ETF、金融ETF、地产ETF等

#### 3.2.2 ETF与个股的异同

| 对比项 | 个股 | 行业ETF |
|--------|------|---------|
| 技术指标分析 | ✅ 均线、MACD、成交量 | ✅ 同个股，相同指标 |
| 基本面分析 | ✅ PE、PB、净利润等 | ❌ ETF无基本面，跳过此项 |
| 政策新闻分析 | ✅ 关联所属行业 | ✅ 关联对应板块 |
| 行业趋势加分 | ✅ 参考所属ETF走势 | ❌ 不重复加分（自身即ETF） |
| 综合评分 | ✅ 满分100分 | ✅ 满分100分（重新分配权重） |
| 推送条件 | ≥70分或<30分 | ≥70分或<30分 |
| 图表 | ✅ K线+MACD | ✅ K线+MACD |
| 同花顺链接 | ✅ 个股行情页 | ✅ ETF行情页 |

#### 3.2.3 ETF综合评分权重（满分100分）

由于ETF没有基本面数据，将基本面权重重新分配：

| 维度 | 权重 | 满分 | 说明 |
|------|------|------|------|
| 技术指标 | 55% | 55分 | 均线、MACD、成交量信号（评分规则同个股） |
| 政策新闻 | 35% | 35分 | 政策对该板块的利好/利空影响 |
| 资金流向 | 10% | 10分 | ETF份额变化反映资金流入/流出 |

#### 3.2.4 ETF资金流向评分细则（满分10分）

| 信号 | 得分 | 条件 |
|------|------|------|
| 份额持续增长 | +8~10 | 近5个交易日ETF份额连续增长 |
| 份额小幅增长 | +4~7 | 近5个交易日ETF份额总体增长 |
| 份额基本持平 | +3 | 近5个交易日ETF份额无明显变化 |
| 份额持续减少 | 0~2 | 近5个交易日ETF份额连续下降 |

### 3.3 技术指标分析

对筛选后的股票计算以下技术指标（基于最近60个交易日数据）：

#### 3.3.1 均线系统（MA）

| 指标 | 计算方式 | 信号含义 |
|------|----------|----------|
| MA5 / MA10 金叉 | MA5 上穿 MA10 | 短期看多信号 |
| MA5 / MA10 死叉 | MA5 下穿 MA10 | 短期看空信号 |
| MA20 / MA60 金叉 | MA20 上穿 MA60 | 中期趋势转多 |
| MA20 / MA60 死叉 | MA20 下穿 MA60 | 中期趋势转空 |
| 股价与MA20关系 | 收盘价 > MA20 | 站上中期均线，偏多 |
| 多头排列 | MA5 > MA10 > MA20 > MA60 | 强势多头信号 |

#### 3.3.2 MACD

| 指标 | 计算方式 | 信号含义 |
|------|----------|----------|
| DIF上穿DEA（金叉） | DIF 从下向上穿越 DEA | 买入信号 |
| DIF下穿DEA（死叉） | DIF 从上向下穿越 DEA | 卖出信号 |
| 零轴上方金叉 | DIF > 0 且 DEA > 0 时金叉 | 强买入信号 |
| MACD柱由负转正 | MACD柱状图由绿转红 | 多方力量增强 |
| 底背离 | 股价创新低，MACD不创新低 | 潜在反转信号 |

#### 3.3.3 成交量

| 指标 | 计算方式 | 信号含义 |
|------|----------|----------|
| 量比 | 当日成交量 / 过去5日平均成交量 | > 1.5 为放量 |
| 放量上涨 | 涨幅 > 0 且 量比 > 1.5 | 资金积极入场 |
| 缩量下跌 | 跌幅 < 0 且 量比 < 0.7 | 抛压减弱，可能企稳 |
| 天量见天价 | 成交量创60日新高 | 可能见顶信号 |

### 3.4 基本面数据

| 指标 | 数据来源 | 用途 |
|------|----------|------|
| 滚动市盈率（PE-TTM） | AKShare | 估值判断 |
| 市净率（PB） | AKShare | 估值辅助 |
| 总市值 | AKShare | 筛选与权重参考 |
| 净利润同比增长率 | AKShare | 盈利能力评估 |
| 营收同比增长率 | AKShare | 成长性评估 |

### 3.5 政策新闻分析

#### 3.5.1 新闻来源

从以下渠道抓取当日财经新闻：

| 来源 | 可信度 | 说明 |
|------|--------|------|
| 新浪财经 | 高 | 主流财经媒体 |
| 东方财富 | 高 | 主流财经平台 |
| 同花顺 | 高 | 主流财经平台 |

#### 3.5.2 新闻处理流程

```
抓取新闻 → 过滤非政策类新闻 → 提取关键词 → 关联行业/板块
→ 结合历史经验评估影响 → 生成新闻摘要 → 关联到对应个股
```

#### 3.5.3 新闻可信度评估

- **高可信**：政府官方发布（国务院、央行、证监会等）、三大证券报
- **中可信**：主流财经媒体（东方财富、同花顺、新浪财经）
- **低可信**：自媒体、论坛传闻（**不纳入分析**）

#### 3.5.4 政策影响分析

结合历史经验，对政策新闻进行影响评估：

| 政策类型 | 历史经验参考 | 影响方向 |
|----------|-------------|----------|
| 降准降息 | 通常利好银行、地产、基建 | 利多 |
| 行业补贴（如新能源） | 短期利好相关板块 | 利多 |
| 加征关税/制裁 | 利空出口依赖型企业 | 利空 |
| IPO暂停 | 短期利好市场情绪 | 利多 |
| 严查配资/杠杆 | 短期利空券商、小盘股 | 利空 |

### 3.6 综合评分体系（满分100分）

#### 3.6.1 评分维度与权重

**个股评分权重：**

| 维度 | 权重 | 满分 | 说明 |
|------|------|------|------|
| 技术指标 | 40% | 40分 | 均线、MACD、成交量信号 |
| 基本面 | 30% | 30分 | PE、PB、净利润增长率等 |
| 政策新闻 | 20% | 20分 | 政策利好/利空影响 |
| 行业趋势 | 10% | 10分 | 所属行业ETF走势参考 |

**ETF评分权重（见3.2.3节）：**

| 维度 | 权重 | 满分 | 说明 |
|------|------|------|------|
| 技术指标 | 55% | 55分 | 均线、MACD、成交量信号 |
| 政策新闻 | 35% | 35分 | 政策对该板块的利好/利空影响 |
| 资金流向 | 10% | 10分 | ETF份额变化反映资金流入/流出 |

#### 3.6.2 技术指标评分细则（满分40分）

| 信号 | 得分 | 条件 |
|------|------|------|
| MA5/10金叉 | +5 | 近3个交易日内发生 |
| MA20/60金叉 | +5 | 近5个交易日内发生 |
| 多头排列 | +5 | MA5 > MA10 > MA20 > MA60 |
| 股价站上MA20 | +3 | 收盘价 > MA20 |
| MACD金叉 | +5 | 近3个交易日内发生 |
| 零轴上方金叉 | +5 | DIF > 0 且 DEA > 0 |
| MACD底背离 | +5 | 近10个交易日内出现 |
| 放量上涨 | +4 | 涨幅 > 0 且 量比 > 1.5 |
| 缩量回调到位 | +3 | 回调至MA20附近且缩量 |
| **扣分项** | | |
| MA5/10死叉 | -5 | 近3个交易日内发生 |
| MACD死叉 | -5 | 近3个交易日内发生 |
| 天量见天价 | -5 | 成交量60日新高且股价高位 |
| 放量下跌 | -4 | 跌幅 > 2% 且 量比 > 2.0 |

#### 3.6.3 基本面评分细则（满分30分）

| 指标 | 得分 | 条件 |
|------|------|------|
| PE-TTM（合理偏低） | +8 | 10 < PE < 30 |
| PE-TTM（偏高） | +3 | 30 ≤ PE < 60 |
| PE-TTM（过高） | 0 | PE ≥ 60 |
| PB（合理） | +5 | 1 < PB < 5 |
| 净利润同比增长 > 20% | +10 | 盈利高增长 |
| 净利润同比增长 0~20% | +5 | 盈利稳定增长 |
| 净利润同比下降 | 0 | 盈利下滑 |
| 营收同比增长 > 15% | +7 | 营收高增长 |
| 营收同比增长 0~15% | +3 | 营收稳定增长 |
| 营收同比下降 | 0 | 营收下滑 |

#### 3.6.4 政策新闻评分细则（满分20分）

| 影响程度 | 得分 | 条件 |
|----------|------|------|
| 直接利好（所属行业） | +15~20 | 政策直接利好该股票所在行业 |
| 间接利好 | +8~14 | 政策间接利好相关产业链 |
| 中性 | +5 | 无明显政策影响 |
| 间接利空 | -5~0 | 政策间接利空相关产业链 |
| 直接利空 | -10~-5 | 政策直接利空该股票所在行业 |

#### 3.6.5 行业趋势评分细则（满分10分，仅个股）

> 仅用于个股评分，ETF自身即为行业代表，不重复加分。

| 信号 | 得分 | 条件 |
|------|------|------|
| 所属行业ETF处于上升趋势 | +8~10 | 行业ETF均线多头排列 |
| 所属行业ETF横盘震荡 | +4~5 | 行业ETF无明显趋势 |
| 所属行业ETF处于下降趋势 | 0~2 | 行业ETF均线空头排列 |

#### 3.6.6 综合判定规则

| 综合评分 | 判定 | 推送行为 |
|----------|------|----------|
| ≥ 70 分 | **高概率盈利** | 推送，标注"建议关注" |
| 50 ~ 69 分 | 观望 | 不推送 |
| 30 ~ 49 分 | 偏空 | 不推送 |
| < 30 分 | **特殊风险** | 推送，标注"风险警示" |

```yaml
workflow_dispatch:
```

### 4.5 环境变量（GitHub Secrets）

| 变量名 | 说明 | 获取方式 |
|--------|------|----------|
| `PUSHPLUS_TOKEN` | PushPlus 推送令牌 | 注册 PushPlus 后获取 |

### 4.6 数据抓取规范

- **随机延迟**：每次 API 请求之间加入 1~5 秒的随机延迟，避免触发频率限制
- **重试机制**：请求失败时最多重试 3 次，间隔递增（2s、4s、8s）
- **异常处理**：数据抓取失败时记录日志，不影响其他股票的分析

## 五、非功能需求

### 5.1 可靠性

- 单只股票数据异常不应影响整体分析流程
- GitHub Actions 执行失败时发送告警通知

### 5.2 可维护性

- 评分权重可通过配置文件调整，无需修改代码
- 新闻来源可扩展，新增来源只需添加配置

### 5.3 安全性

- PushPlus Token 等敏感信息通过 GitHub Secrets 存储
- 不在代码中硬编码任何密钥

### 5.4 免责声明

- 每次推送末尾附带免责声明："以上分析仅供参考，不构成投资建议。投资有风险，入市需谨慎。"

## 六、验收标准
