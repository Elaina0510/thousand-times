## 一、系统简介

A股智能选股分析与推送系统是一套自动化的A股行情分析工具。系统每日定时运行，从公开数据源抓取股票和行业ETF的行情数据，通过**技术指标**、**基本面数据**和**政策新闻**进行综合评分，筛选出高概率盈利或存在特殊风险的标的，生成分析报告并通过微信推送。

### 核心特点

- **宁缺毋滥**：只推送综合评分 ≥ 75 分（建议关注）或 < 45 分（风险警示）的标的
- **多维分析**：技术面 + 基本面 + 政策面 + 行业趋势，四维度综合评判
- **买卖信号**：每个推送标的附带买入/观望/卖出信号和关键价位
- **自动化**：GitHub Actions 每日工作日自动执行，无需人工干预

## 二、快速开始

### 2.1 注册 PushPlus

1. 访问 [PushPlus 官网](https://www.pushplus.plus/)
2. 点击"注册"，使用微信扫码注册
3. 注册完成后，在"个人中心"页面复制你的 **Token**

### 2.2 配置 GitHub 仓库

1. Fork 本项目到你的 GitHub 账号
2. 进入仓库 → Settings → Secrets and variables → Actions
3. 添加以下 Secrets：

| Secret 名称 | 值 | 说明 |
|-------------|------|------|
| `PUSHPLUS_TOKEN` | 你的 PushPlus Token | **必填**，用于微信推送 |
| `LLM_API_URL` | LLM API 地址 | 可选，用于政策新闻分析 |
| `LLM_API_KEY` | LLM API 密钥 | 可选，用于政策新闻分析 |

### 2.3 启用 GitHub Actions

1. 进入仓库 → Actions 标签页
2. 如果提示 "Workflows aren't running"，点击 "I understand my workflows, go ahead and enable them"
3. 系统会在每个工作日北京时间 12:00 自动运行

### 2.4 手动触发

1. 进入仓库 → Actions → "Daily Stock Analysis"
2. 点击 "Run workflow"
3. 等待运行完成（约 15-30 分钟）

### 3.2 单只股票/ETF 信息解读

每只推送的标的包含以下信息：

```
【1】贵州茅台 (600519) — 综合评分：82分
├ 技术信号：MA5/10金叉，MACD零轴上方金叉，放量上涨
├ 基本面：评分25分
├ 政策影响：消费刺激政策利好白酒板块
├ 盈利概率：约 88%
├ 信号区间：🟢 买入区（82分）
├ 关键价位：支撑 1750.00 / 压力 1920.00 / 目标 2016.00 / 止损 1662.50
├ 板块对比：白酒 第3/15（超过80%同行）
├ vs板块ETF：跑赢 2.3%
└ 🔗 同花顺详情：https://stockpage.10jqka.com.cn/600519/
[附K线图 + MACD图]
```

#### 字段说明

| 字段 | 含义 |
|------|------|
| 综合评分 | 多维度加权后的总分（0-100），越高越看好 |
| 技术信号 | 当前识别到的技术指标信号 |
| 基本面 | PE、PB、净利润增长率等财务指标的评分 |
| 政策影响 | 最新政策新闻对该行业的影响 |
| 盈利概率 | 基于评分的盈利概率估算 |
| 信号区间 | 🟢买入区（≥75）/ 🟡观望区（45-74）/ 🔴卖出区（<45） |
| 关键价位 | 支撑位、压力位、目标价、止损价 |
| 板块对比 | 在所属板块内的排名和百分位 |
| vs板块ETF | 相对板块ETF的超额收益 |

### 3.3 信号区间解读

| 信号 | 评分范围 | 建议 |
|------|----------|------|
| 🟢 买入区 | ≥ 75 分 | 多个维度指标看好，可关注 |
| 🟡 观望区 | 45 ~ 74 分 | 信号不明确，建议等待 |
| 🔴 卖出区 | < 45 分 | 多个维度指标看空，注意风险 |

### 3.4 关键价位解读

| 价位 | 计算方式 | 用途 |
|------|----------|------|
| 支撑位 | MA20 与近期低点的加权平均 | 跌到此价位可能反弹 |
| 压力位 | MA20 与近期高点的加权平均 | 涨到此价位可能回调 |
| 目标价 | 压力位 × 1.05 | 突破压力位后的上涨目标 |
| 止损价 | 支撑位 × 0.95 | 跌破支撑位后的止损参考 |

## 四、评分体系详解

### 4.1 个股评分（满分100分）

个股采用**四维度加权模型**：

| 维度 | 权重 | 满分 | 分析内容 |
|------|------|------|----------|
| 技术指标 | 35% | 35分 | MA均线、MACD、成交量信号 |
| 趋势判断 | 25% | 25分 | 行业趋势、政策影响 |
| 量价配合 | 20% | 20分 | 量价关系、资金流向 |
| 基本面 | 20% | 20分 | PE、PB、净利润增长率、营收增长率 |

#### 技术指标评分细则

| 信号 | 得分 | 条件 |
|------|------|------|
| MA5/10金叉 | +5 | 近3个交易日内发生 |
| MA20/60金叉 | +5 | 近5个交易日内发生 |
| 多头排列 | +5 | MA5 > MA10 > MA20 > MA60 |
| 股价站上MA20 | +3 | 收盘价 > MA20 |
| MACD金叉 | +5 | 近3个交易日内发生 |
| 零轴上方金叉 | +5 | DIF > 0 且 DEA > 0 |
| MACD底背离 | +5 | 近10个交易日内出现 |
| 放量上涨 | +4 | 涨幅 > 0 且 量比 > 1.5 |
| 缩量回调到位 | +3 | 回调至MA20附近且缩量 |

#### 扣分项

| 信号 | 扣分 | 条件 |
|------|------|------|
| MA5/10死叉 | -5 | 近3个交易日内发生 |
| MACD死叉 | -5 | 近3个交易日内发生 |
| 天量见天价 | -5 | 成交量60日新高且股价高位 |
| 放量下跌 | -4 | 跌幅 > 2% 且 量比 > 2.0 |

#### 基本面评分细则

| 指标 | 得分 | 条件 |
|------|------|------|
| PE-TTM（合理偏低） | +8 | 10 < PE < 30 |
| PE-TTM（偏高） | +3 | 30 ≤ PE < 60 |
| PE-TTM（过高） | 0 | PE ≥ 60 |
| PB（合理） | +5 | 1 < PB < 5 |
| 净利润同比增长 > 20% | +10 | 盈利高增长 |
| 净利润同比增长 0~20% | +5 | 盈利稳定增长 |
| 净利润同比下降 | 0 | 盈利下滑 |
| 营收同比增长 > 15% | +7 | 营收高增长 |
| 营收同比增长 0~15% | +3 | 营收稳定增长 |
| 营收同比下降 | 0 | 营收下滑 |

### 4.2 ETF评分（满分100分）

ETF没有基本面数据，采用**三维度模型**：

| 维度 | 权重 | 满分 | 分析内容 |
|------|------|------|----------|
| 技术指标 | 55% | 55分 | 均线、MACD、成交量信号（同个股） |
| 政策新闻 | 35% | 35分 | 政策对该板块的利好/利空影响 |
| 资金流向 | 10% | 10分 | ETF份额变化反映资金流入/流出 |

#### 资金流向评分细则

| 信号 | 得分 | 条件 |
|------|------|------|
| 份额持续增长 | +8~10 | 近5个交易日ETF份额连续增长 |
| 份额小幅增长 | +4~7 | 近5个交易日ETF份额总体增长 |
| 份额基本持平 | +3 | 近5个交易日ETF份额无明显变化 |
| 份额持续减少 | 0~2 | 近5个交易日ETF份额连续下降 |

## 五、推送标的说明

### 5.1 推送条件

系统只在以下情况推送：

1. **综合评分 ≥ 75 分**：标注为"建议关注"，附带买入信号
2. **综合评分 < 45 分**：标注为"风险警示"，附带卖出信号

评分在 45~74 分之间的标的**不会推送**（信号不明确）。

### 5.2 推送内容

每只推送标的包含：

- 技术信号摘要（金叉/死叉、多头/空头排列等）
- 基本面评分（仅个股）
- 政策影响摘要
- 盈利概率估算
- 买卖信号区间和关键价位
- 板块对比排名（仅个股）
- 同花顺行情链接
- K线图 + MACD 图

### 5.3 无推荐场景

当日无任何标的满足推送条件时，系统会推送"今日无推荐"消息，附带政策要闻摘要。

## 六、常见问题

### Q1: 为什么没有收到推送？

可能原因：
1. **PUSHPLUS_TOKEN 未配置**：检查 GitHub Secrets 是否正确添加
2. **Token 过期**：登录 PushPlus 官网检查 Token 是否有效
3. **当日无推荐**：所有标的评分在观望区间，系统推送"今日无推荐"
4. **GitHub Actions 执行失败**：进入 Actions 页面查看运行状态

### Q2: 推送的时间是什么时候？

- **自动推送**：每个工作日北京时间 12:00（UTC 04:00）
- **手动触发**：在 GitHub Actions 页面手动运行，完成后立即推送

### Q3: 为什么有些股票/ETF没有分析到？

- 系统只分析市值 ≥ 20亿、非ST、PE > 0、上市 ≥ 3个月的股票
- ETF 只分析预设池中的 11 只主流 ETF
- 单只股票/ETF 数据获取失败时会被跳过

### Q4: 盈利概率准确吗？

盈利概率是基于综合评分的**统计估算**，不是精确预测。评分 70 分对应约 88% 的盈利概率，但实际投资结果受多种因素影响。**请勿将推送结果作为唯一投资依据**。

### Q5: 如何自定义分析的股票/ETF？

- **修改 ETF 池**：编辑 `src/config.py` 中的 `DEFAULT_ETF_POOL` 列表
- **修改股票池大小**：编辑 `src/config.py` 中的 `FilterConfig.pool_size`（默认 200）
- **修改评分权重**：编辑 `src/config.py` 中的 `ScoreWeightConfig` 和相关 dataclass

### Q6: 如何查看历史分析报告？

- **GitHub Actions 日志**：进入 Actions → 选择历史运行 → 查看日志
- **本地日志**：运行后在 `logs/` 目录下生成 `analysis.log` 和 `report_<日期>.md`
